# 场景定义：Bug 修复

## 典型特征

- 针对特定问题的修复
- 通常范围较窄，涉及少量文件
- 需要根因分析
- 需要回归测试

## 常见类型

1. **逻辑错误**
   - 条件判断错误
   - 边界条件处理

2. **数据问题**
   - 脏数据清理
   - 数据迁移修复

3. **性能问题**
   - N+1 查询
   - 内存泄漏
   - 死锁

4. **安全问题**
   - 注入漏洞
   - XSS/CSRF
   - 权限绕过

## 常见决策点

1. **修复策略**
   - 治标 vs 治本
   - 快速修复 vs 彻底重构

2. **数据修复**
   - 是否需要数据迁移
   - 影响范围评估

3. **发布策略**
   - Hotfix vs 常规发布
   - 回滚准备

## Triage 策略

- **Level 判定**: 通常为 Level 2-3
  - 简单修复 → Level 2
  - 需要根因分析/数据修复 → Level 3
- **关键信息**: 影响范围、紧急程度、验证方案
- **常见风险**: 修复不彻底、引入新问题

## 信息保留重点

1. Bug 根因（一句话）
2. 影响范围（用户/数据）
3. 修复方案概要
4. 验证方法
5. 紧急程度
